# 🎬 Рекомендатель аниме по MyAnimeList

Веб-приложение: вводишь свой ник на [MyAnimeList](https://myanimelist.net),
программа берёт список просмотренного и подбирает похожие тайтлы.

## Как работает

- **Список просмотренного** берётся через публичный эндпоинт `load.json` сайта MAL
  (по нику, без регистрации — профиль должен быть **публичным**).
- **Рекомендации** считаются гибридно:
  1. *Сигнал MAL* — для любимых тайтлов берутся рекомендации MyAnimeList
     («кто смотрел это — советует то») через [Jikan API](https://jikan.moe),
     голоса взвешиваются твоими оценками.
  2. *Сигнал жанров* — строится профиль вкусов по жанрам (с учётом оценок),
     кандидаты ранжируются по близости к нему.
  - Итог: `alpha * сигнал_MAL + (1 - alpha) * сигнал_жанров` (ползунок в интерфейсе).
- **Отметка «уже смотрел»** (опционально) — после входа через MyAnimeList на
  каждой карточке появляется кнопка «✅ Смотрел» и выбор оценки; тайтл
  записывается в твой список MAL как `completed` через официальный
  [MAL API v2](https://myanimelist.net/apiconfig/references/api/v2) (OAuth2).

## Структура

| Файл | Назначение |
|------|------------|
| `mal_client.py` | Загрузка списка с MAL и обращения к Jikan API |
| `recommender.py` | Гибридная логика рекомендаций |
| `app.py` | Веб-интерфейс на Streamlit |

## Запуск

```powershell
# из папки проекта, на Python 3.13
venv313\Scripts\streamlit run app.py
```

Откроется вкладка в браузере. Введи ник слева, настрой ползунки, жми
«Подобрать рекомендации».

## Установка зависимостей (если нужно с нуля)


## Запись в MyAnimeList (вход через OAuth2)

Чтобы отмечать тайтлы просмотренными прямо в своём списке MAL:

1. Зарегистрируй приложение на <https://myanimelist.net/apps>:
   - **App Type:** `web`
   - **App Redirect URL:** `http://localhost:8501/` (точно так, со слэшем)
2. Скопируй `.env.example` в `.env` и впиши `MAL_CLIENT_ID` и `MAL_CLIENT_SECRET`.
3. Запусти приложение, нажми слева **«Войти через MyAnimeList»**, разреши доступ.
4. На карточках появятся кнопки **«✅ Смотрел»** с выбором оценки — клик пишет
   статус `completed` в твой список через официальный API.

Access-токен живёт час и обновляется автоматически (refresh-токен — месяц).
Токен хранится только в сессии браузера (на диск не пишется), поэтому на общем
сервере посетители не делят один вход — каждый авторизуется сам.

## Публичный хостинг (Streamlit Community Cloud)

Чтобы сервисом могли пользоваться другие, его нужно развернуть на хостинге —
например, бесплатно на <https://share.streamlit.io>:

1. Залей репозиторий на GitHub (уже сделано) и подключи его на share.streamlit.io,
   указав `app.py` как главный файл.
2. В настройках приложения MAL (<https://myanimelist.net/apiconfig>) добавь
   **второй App Redirect URL** — публичный адрес приложения, например
   `https://твой-домен.streamlit.app/` (несколько URL разделяются переносом строки).
3. В разделе **Secrets** на Streamlit Cloud задай (см. `.streamlit/secrets.toml.example`):
   ```toml
   MAL_CLIENT_ID = "..."
   MAL_CLIENT_SECRET = "..."
   MAL_REDIRECT_URI = "https://твой-домен.streamlit.app/"
   ```
4. Каждый посетитель входит под своим аккаунтом MAL — токены изолированы по сессиям.

Замечание: лимиты Jikan (~3 запроса/сек) общие на всех пользователей сервиса,
поэтому при одновременной нагрузке подбор будет идти медленнее.

## Замечания

- Jikan ограничивает частоту запросов (~3/сек), поэтому подбор занимает
  10–30 секунд — это нормально, идёт прогресс-бар.
- Профиль на MAL должен быть публичным, иначе список не загрузится.
